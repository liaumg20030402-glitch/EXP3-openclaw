\section{实验题目}

使用 Agent + Skills 完成化学任务：设计并应用一个面向\textbf{化学反应配平与化学计量分析}的
Agent Skill（\codepath{reaction-stoichiometry}），并在 OpenClaw + 飞书环境中通过自然语言
驱动智能体调用该 Skill 完成化学计算任务。

\section{实验目的}

\begin{enumerate}
    \item 理解 Agent（通用智能体）与 Skill（可复用能力模块）的协作机制，体会 Skill 与一次性
          Prompt 的本质区别：Skill 把专业流程、规则约束与工具调用方式封装成可按需加载、可复用、
          可稳定执行的能力。
    \item 完成 OpenClaw 的安装与配置，并将其接入飞书，实现通过飞书与智能体对话。
    \item 仿照课程示例仓库 \codepath{exp3-agent-chem-skills}，\textbf{独立编写一个与示例不同的
          Skill}，完成一类新的化学任务。
    \item 在 OpenClaw 中利用该 Skill，通过自然语言提示完成化学相关任务并获得结构化结果。
\end{enumerate}

\section{编写 Agent Skills}

\subsection{设计目标与选题}

本实验设计了一个全新主题的 Skill —— \textbf{反应配平与化学计量分析}
（\codepath{reaction-stoichiometry}）。

在设计中特别考虑了一个核心问题：\textit{这个任务是否``普通大语言模型也能直接回答''}。
为使 Skill 真正体现价值，最终把它扩展为一个\textbf{确定性计算工具集}，做的是裸 LLM
\textbf{做不可靠、甚至做不到}的事：

\begin{enumerate}
    \item \textbf{精确配平}：用\textbf{有理数线性代数}（分数精确运算）求解配平系数，避免 LLM
          在复杂氧化还原反应中``幻觉''出错误系数。
    \item \textbf{确定性计算}：限制试剂、理论产量、百分产率、原子经济性 / E-factor、反应焓
          $\Delta H$ 全部由脚本精确算出，结果可复现、可审计。
    \item \textbf{生成真实文件}：输出 JSON 结果文件与产量图（PNG，环境无 matplotlib 时自动
          回退为 SVG）—— 纯文本模型无法产出文件。
\end{enumerate}

\subsection{核心脚本：算法实现}

核心脚本 \codepath{stoichiometry_helper.py} 仅依赖 Python 标准库，由\textbf{四个子命令}组成，
它们共享底层的化学式解析、摩尔质量计算与配平模块。脚本通过 \codepath{argparse} 组织子命令：

\begin{lstlisting}
sub = p.add_subparsers(dest="cmd", required=True)
sub.add_parser("balance",   ...)   # --equation [--output]
sub.add_parser("stoich",    ...)   # --equation --given [--actual --target --plot --output]
sub.add_parser("empirical", ...)   # --combustion --sample | --percent  [--molar-mass --output]
sub.add_parser("thermo",    ...)   # --equation [--dhf --output]
\end{lstlisting}

每个子命令成功时都打印若干 \codepath{[RESULT] key=value} 行和一行以 \codepath{[JSON]} 开头的
完整结果，便于 Agent 解析；出错时打印 \codepath{[ERROR] ...} 并返回非零退出码。下面依次说明
公共模块与四个子功能的算法。

\subsubsection{公共模块：化学式解析与摩尔质量}

\codepath{parse_formula} 用一个支持栈的递归下降解析器，把化学式解析为``元素 $\to$ 原子数''
字典，支持\textbf{嵌套括号/方括号、下标、前置系数、水合物点号}（如 \codepath{CuSO4·5H2O}，
分隔符 \codepath{·•*.} 均可），并忽略状态符 \codepath{(s)/(l)/(g)/(aq)} 与离子电荷。摩尔质量由
内置 IUPAC 标准原子量表 $A_{el}$ 求和得到：
\[
    M = \sum_{el} n_{el}\, A_{el}.
\]

\subsubsection{子功能一：方程式配平（balance）}

配平的本质是求解元素守恒方程组的零空间。把每个物种解析为元素向量后，构造
``元素 $\times$ 物种''系数矩阵 $\mathbf{A}$（反应物为正、产物为负），配平即求满足
$\mathbf{A}\mathbf{x}=\mathbf{0}$ 的最小正整数解 $\mathbf{x}$。脚本用\textbf{分数精确运算}
（\codepath{fractions.Fraction}）做行最简化（RREF）求零空间基向量，再清分母、约去最大公约数
化为最小正整数系数，从根本上避免浮点误差与``幻觉''系数：

\begin{lstlisting}
def balance_equation(reactants, products):
    species = reactants + products
    nr = len(reactants)
    parsed = [parse_formula(s) for s in species]
    elements = sorted({el for p in parsed for el in p})
    matrix = []
    for el in elements:                               # element x species matrix
        matrix.append([Fraction(p.get(el, 0) if i < nr else -p.get(el, 0))
                       for i, p in enumerate(parsed)])
    basis = _nullspace_basis(matrix, len(species))    # rational null space
    if len(basis) == 0:
        raise ValueError("Equation cannot be balanced ...")
    if len(basis) > 1:
        raise ValueError("Equation is underdetermined ...")
    vec = basis[0]
    lcm = 1                                           # clear denominators -> ints
    for f in vec:
        lcm = lcm * f.denominator // math.gcd(lcm, f.denominator)
    ints = [int(f * lcm) for f in vec]
    ...                                               # divide by gcd -> smallest ints
    return ints[:nr], ints[nr:]
\end{lstlisting}

当零空间维数为 0（无法配平）或大于 1（含多个独立反应）时，脚本报错并提示用户，而不是
强行给出可疑结果。

\subsubsection{子功能二：化学计量与绿色化学指标（stoich）}

在配平得到系数 $\nu_i$ 后，把每个给定反应物用量换算为物质的量 $n_i$，以\textbf{反应进度}
$\xi$ 衡量反应推进程度，并由此确定限制试剂：
\[
    \xi = \min_{i\,\in\,\text{已给反应物}} \frac{n_i}{\nu_i},
    \qquad \text{限制试剂} = \arg\min_i \frac{n_i}{\nu_i}.
\]
各产物 $j$ 的理论产量与各反应物的剩余量为
\[
    n_{\text{prod},j} = \nu_j\,\xi,\qquad
    \text{剩余}_i = n_i - \nu_i\,\xi .
\]
脚本还计算两个绿色化学指标——\textbf{原子经济性}（目标产物 $t$ 占全部反应物质量的比例）与
理论 \textbf{E-factor}（废物质量 / 目标产物质量）：
\[
    \mathrm{AE} = \frac{\nu_t M_t}{\sum_i \nu_i M_i}\times100\%,
    \qquad
    E = \frac{\sum_i \nu_i\xi M_i - \nu_t\xi M_t}{\nu_t\xi M_t}.
\]
若用户额外给出某产物的\textbf{实际产量} $m_\text{actual}$，则计算百分产率
$m_\text{actual}/(\nu\xi M)\times100\%$。核心逻辑如下：

\begin{lstlisting}
# extent of reaction = min over given reactants of (moles / coefficient)
xi = min(given_moles[n] / coeff[n] for n in reactants if n in given_moles)
limiting = [n for n in given_moles if n in reactants
            and abs(given_moles[n]/coeff[n] - xi) < 1e-12]
atom_economy = 100 * coeff[target]*mw[target] / sum(coeff[n]*mw[n] for n in reactants)
for n in products:                       # theoretical yield of each product
    theoretical_g[n] = coeff[n] * xi * mw[n]
\end{lstlisting}

该子命令总是额外打印一个 \codepath{[CHART]} \textbf{文本柱状图}（见第~\ref{sec:problems}~节问题二），
并可用 \codepath{--plot} 把图额外存为 PNG/SVG 文件。

\subsubsection{子功能三：实验式 / 分子式反推（empirical）}

支持两种输入。\textbf{燃烧分析}：含 C、H（可能含 O）的有机物完全燃烧，由 $\mathrm{CO_2}$、
$\mathrm{H_2O}$ 质量得到 C、H 物质的量，O 由样品质量差减求得：
\[
    n_C = \frac{m_{\mathrm{CO_2}}}{M_{\mathrm{CO_2}}},\quad
    n_H = \frac{2\,m_{\mathrm{H_2O}}}{M_{\mathrm{H_2O}}},\quad
    n_O = \frac{m_\text{sample} - n_C A_C - n_H A_H}{A_O}.
\]
\textbf{元素质量百分比}：以 100\,g 为基准，$n_{el} = w_{el}/A_{el}$。随后把各物质的量除以最小值，
并乘以 $1\!\sim\!8$ 的整数倍直到全部接近整数，得到\textbf{实验式}下标。若给定摩尔质量 $M$，
则按倍数 $k=\mathrm{round}(M/M_\text{emp})$ 放大为\textbf{分子式}：

\begin{lstlisting}
molC = m_CO2 / M("CO2")                 # 1 C per CO2
molH = 2 * m_H2O / M("H2O")             # 2 H per H2O
mO   = sample_g - molC*A("C") - molH*A("H")        # O by difference
subs = integer_ratios({"C": molC, "H": molH, "O": mO/A("O")})   # -> CH2O
mult = round(molar_mass_given / formula_mass(subs))             # x6 -> C6H12O6
\end{lstlisting}

\subsubsection{子功能四：反应焓计算（thermo）}

先配平方程式，再用内置\textbf{标准生成焓}表 $\Delta H_f^\circ$（kJ/mol，元素单质为 0）按
\textbf{盖斯定律}计算反应焓：
\[
    \Delta H_\text{rxn} = \sum_{\text{产物}} \nu\,\Delta H_f^\circ
                        - \sum_{\text{反应物}} \nu\,\Delta H_f^\circ ,
\]
并据 $\Delta H<0$ / $>0$ 判定放热 / 吸热。若某物种不在内置表中，脚本报错并提示用
\codepath{--dhf} 参数补充其生成焓，避免编造数据。

\begin{lstlisting}
dH = (sum(nu * dHf[p] for p, nu in products)      # Hess's law
      - sum(nu * dHf[r] for r, nu in reactants))
\end{lstlisting}

内置 $\Delta H_f^\circ$ 默认采用常规标准态（$\mathrm{H_2O}$ 取液态；金属、氧化物、盐取固态；
气体按写法），报告中会注明该假设。

\subsection{本地自测}

\begin{lstlisting}

\end{lstlisting}

\begin{lstlisting}
$ python stoichiometry_helper.py empirical \
    --combustion "CO2=0.352g, H2O=0.144g" --sample 0.240g --molar-mass 180.16

[RESULT] molecular_formula=C6H12O6

$ python stoichiometry_helper.py thermo --equation "CH4 + O2 -> CO2 + H2O"
[RESULT] balanced=CH4 + 2 O2 -> CO2 + 2 H2O

\end{lstlisting}

该案例完整体现了 Skill 的价值：配平结果精确、化学计量与产率确定可复现，并\textbf{产出了真实的
结果文件与图表}——这些都是普通对话式 LLM 难以可靠完成或根本无法完成的。

\section{遇到的问题及解决方法}
\label{sec:problems}

\textbf{问题一：Skill 选题过于简单。} 最初版本仅做``配平''，而这是普通 LLM 也能回答的问题，
未能体现 Skill 价值。\textit{解决}：将 Skill 扩展为四合一确定性计算工具（配平、化学计量与绿色
指标、实验式反推、反应焓），用有理数精确运算保证正确性，并产出文件工件。

\textbf{问题二：GLM-5.1 回传图片到飞书失败（``media failed''）。} 脚本用 matplotlib 生成的
PNG 在飞书中无法作为图片消息正常显示。\textit{解决}：让 \codepath{stoich} \textbf{总是}在文本
输出中绘制 ASCII 文本柱状图（任何模型/平台都能显示），PNG/SVG 仅作为\textbf{保存到磁盘}的
文件；并在 SKILL.md 中明确要求``用文本柱状图展示、不要把图片当聊天消息发送，但要列出文件
路径''。实测虚拟机中无 matplotlib，脚本按设计\textbf{自动回退生成 SVG}，文件路径正常返回。


\section{总结}

本实验完整走通了``安装 OpenClaw $\to$ 接入飞书 $\to$ 编写 Skill $\to$ 自然语言驱动完成化学
任务''的闭环，并独立设计了一个与课程示例不同的 \codepath{reaction-stoichiometry} Skill。

通过本实验，我对 Agent + Skills 的机制有了更深理解：

\begin{enumerate}
    \item \textbf{Skill 不等于 Prompt}。Skill 把专业流程、规则约束、工具脚本与输出格式封装成
          可复用模块，由 frontmatter 的描述驱动 Agent\textbf{按需自动路由与加载}，用户只需用
          自然语言描述任务，无需关心 Skill 名称或调用细节。
    \item \textbf{好的 Skill 应做 LLM 做不好的事}。把``可能算错''的工作交给确定性脚本（精确
          配平、化学计量、反应焓），把``无法完成''的工作变为可能（生成文件、图表），才真正
          放大了智能体的能力，而非重复 LLM 已能做的事。
    \item \textbf{工程鲁棒性很重要}。零依赖、跨环境兼容、对平台能力（如图片回传）的退化处理，
          决定了 Skill 能否在真实环境中稳定演示与复用。
\end{enumerate}
