\section{实验题目}

使用 Agent + Skills 完成化学任务：设计并应用一个面向\textbf{化学反应配平与化学计量分析}的
Agent Skill（\codepath{reaction-stoichiometry}），并在 OpenClaw + 飞书环境中通过自然语言
驱动智能体调用该 Skill 完成化学计算任务。

\section{实验目的}

\begin{enumerate}
    \item 理解 Agent（通用智能体）与 Skill（可复用能力模块）的协作机制，体会 Skill 与一次性
          Prompt 的本质区别：Skill 把专业流程、规则约束与工具调用方式封装成可按需加载、可复用、
          可稳定执行的能力。
    \item 完成 OpenClaw 的安装与配置，并将其接入飞书，实现通过飞书与智能体对话。
    \item 仿照课程示例仓库 \codepath{exp3-agent-chem-skills}，\textbf{独立编写一个与示例不同的
          Skill}，完成一类新的化学任务。
    \item 在 OpenClaw 中利用该 Skill，通过自然语言提示完成化学相关任务并获得结构化结果。
\end{enumerate}

\section{编写 Agent Skills}

\subsection{设计目标与选题}

课程示例仓库提供了三个实验级 Skill：化学品安全报告（联网检索 SDS/GHS）、分子 3D Profile
（PubChem + RDKit + 3D 构象）、xTB 半经验量子化学优化工作流。为体现``与示例不同''的要求，
本实验设计了一个全新主题的 Skill —— \textbf{反应配平与化学计量分析}
（\codepath{reaction-stoichiometry}）。

在设计中特别考虑了一个核心问题：\textit{这个任务是否``普通大语言模型也能直接回答''}。
为使 Skill 真正体现价值，最终把它扩展为一个\textbf{确定性计算工具集}，做的是裸 LLM
\textbf{做不可靠、甚至做不到}的事：

\begin{enumerate}
    \item \textbf{精确配平}：用\textbf{有理数线性代数}（分数精确运算）求解配平系数，避免 LLM
          在复杂氧化还原反应中``幻觉''出错误系数。
    \item \textbf{确定性计算}：限制试剂、理论产量、百分产率、原子经济性 / E-factor、反应焓
          $\Delta H$ 全部由脚本精确算出，结果可复现、可审计。
    \item \textbf{生成真实文件}：输出 JSON 结果文件与产量图（PNG，环境无 matplotlib 时自动
          回退为 SVG）—— 纯文本模型无法产出文件。
\end{enumerate}

该 Skill 的核心功能\textbf{仅依赖 Python 标准库}（\codepath{fractions}），不联网、不依赖
RDKit，因此演示时不会因网络或第三方依赖缺失而失败，可用 \codepath{python}、
\codepath{python3} 或 \codepath{uv run} 任一方式运行。

\subsection{核心脚本：精确配平算法}

配平的本质是求解元素守恒方程组的零空间。脚本把每个物种解析为``元素 $\to$ 原子数''的向量，
构造``元素 $\times$ 物种''系数矩阵（反应物为正、产物为负），再用\textbf{分数精确运算}做行最简化
（RREF）求零空间基向量，最后清分母、取最大公约数化为最小正整数系数。核心片段如下：

\begin{lstlisting}
def balance_equation(reactants, products):
    species = reactants + products
    nr = len(reactants)
    parsed = [parse_formula(s) for s in species]
    elements = sorted({el for p in parsed for el in p})
    matrix = []
    for el in elements:
        matrix.append([Fraction(p.get(el, 0) if i < nr else -p.get(el, 0))
                       for i, p in enumerate(parsed)])
    basis = _nullspace_basis(matrix, len(species))   # rational null space
    if len(basis) == 0:
        raise ValueError("Equation cannot be balanced ...")
    if len(basis) > 1:
        raise ValueError("Equation is underdetermined ...")
    vec = basis[0]
    lcm = 1                                           # clear denominators -> ints
    for f in vec:
        lcm = lcm * f.denominator // math.gcd(lcm, f.denominator)
    ints = [int(f * lcm) for f in vec]
    ...                                               # divide by gcd -> smallest ints
    return ints[:nr], ints[nr:]
\end{lstlisting}

\codepath{parse_formula} 支持嵌套括号、下标、水合物点号（如 \codepath{CuSO4·5H2O}）与状态符
\codepath{(s)/(l)/(g)/(aq)}；摩尔质量与生成焓使用内置标准数据表。脚本通过子命令组织（节选）：

\begin{lstlisting}
sub = p.add_subparsers(dest="cmd", required=True)
sub.add_parser("balance",   ...)   # --equation [--output]
sub.add_parser("stoich",    ...)   # --equation --given [--actual --target --plot --output]
sub.add_parser("empirical", ...)   # --combustion --sample | --percent  [--molar-mass --output]
sub.add_parser("thermo",    ...)   # --equation [--dhf --output]
\end{lstlisting}

脚本成功时打印若干 \codepath{[RESULT] key=value} 行和一行以 \codepath{[JSON]} 开头的完整结果，
便于 Agent 解析；\codepath{stoich} 还总是打印 \codepath{[CHART]} 文本柱状图。

\subsection{本地自测}

注册前在本地对四个子命令做了验证，结果均正确。例如复杂氧化还原反应的配平（裸 LLM 易算错）：

\begin{lstlisting}
$ python stoichiometry_helper.py balance \
    --equation "KMnO4 + HCl -> KCl + MnCl2 + H2O + Cl2"
[RESULT] balanced=2 KMnO4 + 16 HCl -> 2 KCl + 2 MnCl2 + 8 H2O + 5 Cl2
\end{lstlisting}

燃烧分析反推分子式（葡萄糖）与甲烷燃烧反应焓：

\begin{lstlisting}
$ python stoichiometry_helper.py empirical \
    --combustion "CO2=0.352g, H2O=0.144g" --sample 0.240g --molar-mass 180.16
[RESULT] empirical_formula=CH2O
[RESULT] molecular_formula=C6H12O6

$ python stoichiometry_helper.py thermo --equation "CH4 + O2 -> CO2 + H2O"
[RESULT] balanced=CH4 + 2 O2 -> CO2 + 2 H2O
[RESULT] dH_rxn_kJ=-890.5          # matches textbook -890.4 kJ/mol
\end{lstlisting}

\subsection{任务与提示}

在飞书中用\textbf{自然语言}（无需说出 Skill 名称）向智能体发送如下提示：

\begin{quote}
反应 C2H6 + O2 -> CO2 + H2O。我有 10\,g 乙烷和 40\,g 氧气，实际只得到了 25\,g CO2。
请配平方程式，判断谁是限制试剂，算出 CO2、H2O 的理论产量、氧气剩余量、CO2 的百分产率

\end{quote}

\subsection{结果解读}

智能体返回的关键结果如下，与本地计算完全一致：

\begin{itemize}
    \item \textbf{配平方程式}：$2\,\mathrm{C_2H_6} + 7\,\mathrm{O_2} \rightarrow
          4\,\mathrm{CO_2} + 6\,\mathrm{H_2O}$；
    \item \textbf{限制试剂}：$\mathrm{C_2H_6}$（乙烷），反应进度 $\xi = 0.1663$\,mol；
    \item \textbf{原子经济性} 61.96\%，理论 \textbf{E-factor} 0.614；
    \item \textbf{百分产率}：$\mathrm{CO_2}$ 85.41\%（实际 25.00\,g / 理论 29.27\,g）；
    \item $\mathrm{CO_2}$、$\mathrm{H_2O}$ 理论产量分别为 29.27\,g、17.97\,g，氧气剩余 2.76\,g。
\end{itemize}

\section{遇到的问题及解决方法}
\label{sec:problems}

\textbf{问题一：Skill 选题过于简单。} 最初版本仅做``配平''，而这是普通 LLM 也能回答的问题，
未能体现 Skill 价值。\textit{解决}：将 Skill 扩展为四合一确定性计算工具（配平、化学计量与绿色
指标、实验式反推、反应焓），用有理数精确运算保证正确性，并产出文件工件。

\textbf{问题二：GLM-5.1 回传图片到飞书失败（``media failed''）。} 脚本用 matplotlib 生成的
PNG 在飞书中无法作为图片消息正常显示。\textit{解决}：让 \codepath{stoich} \textbf{总是}在文本
输出中绘制 ASCII 文本柱状图（任何模型/平台都能显示），PNG/SVG 仅作为\textbf{保存到磁盘}的
文件；并在 SKILL.md 中明确要求``用文本柱状图展示、不要把图片当聊天消息发送，但要列出文件
路径''。实测虚拟机中无 matplotlib，脚本按设计\textbf{自动回退生成 SVG}，文件路径正常返回。


\section{总结}

本实验完整走通了``安装 OpenClaw $\to$ 接入飞书 $\to$ 编写 Skill $\to$ 自然语言驱动完成化学
任务''的闭环，并独立设计了一个与课程示例不同的 \codepath{reaction-stoichiometry} Skill。

通过本实验，我对 Agent + Skills 的机制有了更深理解：

\begin{enumerate}
    \item \textbf{Skill 不等于 Prompt}。Skill 把专业流程、规则约束、工具脚本与输出格式封装成
          可复用模块，由 frontmatter 的描述驱动 Agent\textbf{按需自动路由与加载}，用户只需用
          自然语言描述任务，无需关心 Skill 名称或调用细节。
    \item \textbf{好的 Skill 应做 LLM 做不好的事}。把``可能算错''的工作交给确定性脚本（精确
          配平、化学计量、反应焓），把``无法完成''的工作变为可能（生成文件、图表），才真正
          放大了智能体的能力，而非重复 LLM 已能做的事。
    \item \textbf{工程鲁棒性很重要}。零依赖、跨环境兼容、对平台能力（如图片回传）的退化处理，
          决定了 Skill 能否在真实环境中稳定演示与复用。
\end{enumerate}
